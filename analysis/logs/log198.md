# Log198 — SmallFastDronev1 Indoor Loiter Crash

**Vehicle:** SmallFastDronev1 (BF_X quad, V4.6.3v2-SFD e0cb6544)
**Mode:** Loiter (optical flow + rangefinder)
**Duration:** ~22 seconds airborne (armed 135.2s, takeoff 142.8s, emergency land 164.3s)
**Outcome:** Progressive backward lean, emergency landing

## Root Cause Chain

```
Baro reads 4m too low (THST_SCALE=0 + thermal + ground effect)
    ↓
EKF terrain offset jumps to +4.1m at takeoff (baro vs rangefinder mismatch)
    ↓
With EK3_RNG_USE_HGT=3.0, HAGL = EKF_baro_height - terrain_offset
    ↓
HAGL is 2–4x actual height during 15-second convergence period
    ↓
Optical flow velocity = angular_rate × HAGL → velocities amplified 2–4x
    ↓
Loiter controller fights phantom velocity → commands massive backward pitch
    ↓
Progressive oscillation → crash
```

## Detailed Analysis

### 1. Baro State Before Takeoff

BAlt was already -0.81m before motors started (vehicle sitting on ground). This is
residual thermal drift from the previous flight (log197 cooled the baro from 40.5°C
to 32°C; log198 started at 33.5°C).

With motors at idle (135–142s), BAlt dropped further to -2.4m from ground effect pressure.

### 2. Terrain Offset Convergence Failure

XKF5 terrain offset timeline:
- Takeoff (142.8s): offset = +4.1m (baro says 4m higher than rangefinder)
- +5s (147.8s): offset = +2.5m (slowly converging)
- +10s (152.8s): offset = +0.5m (near zero)
- +15s (157.8s): offset = -1.25m (overshoot!)

During the 15-second convergence, HAGL/rangefinder ratio peaked at 3.8x:
- Rangefinder: 0.4m actual height
- HAGL reported: 1.5m (3.8x too high)

### 3. Flow Velocity Amplification

XKF1 velocity showed persistent VE = -1.5 to -2.3 m/s throughout the flight.
This phantom velocity came from flow angular rates being scaled by the inflated HAGL.

### 4. ATT Data — Progressive Instability

| Phase | Pitch Range | Roll Range |
|-------|------------|------------|
| 143–148s | ±1° | ±2° |
| 148–155s | ±5° | ±3° |
| 155–160s | ±9° | ±5° |
| 160–164s | **±50°** | ±15° |

### 5. Ground Effect Timing

GND_EFFECT_COMP was active with TKOFF_GNDEFF_ALT=0.5m. However, since the EKF thought
the vehicle was at 2.05m (XKF1 PD), the 0.5m threshold was exceeded within milliseconds.
Ground effect protection lasted a fraction of a second — far too short to help.

With TKOFF_GNDEFF_TMO=0 (no timeout), there was no minimum duration guarantee.

### 6. Z-Bias Learning

XKF2 AccZ bias remained stable throughout (-0.150 → -0.155), confirming the Z-bias
learning code was NOT contributing to the crash. The ground effect inhibition worked
correctly during the initial phase.

INS_ACC_VRFB_Z learned to -0.066 during the flight — wrong sign vs the +0.089 VRF
estimated from log197. This is because the learning tracked baro thermal drift (which
was amplified by the upside-down TCAL) rather than actual vibration rectification.

## Key Lessons

1. **EK3_RNG_USE_HGT=3.0 is dangerous with large baro errors** — the terrain offset
   feedback loop amplifies errors rather than correcting them
2. **Ground effect protection needs a timeout** — altitude-based clearing fails when
   the altitude estimate itself is wrong
3. **Baro thermal state persists between flights** — log198 started with 0.81m baro
   error inherited from log197's cooling
4. **Z-bias learning needs clean height reference** — with thermal-contaminated baro,
   the learning converges to the wrong value
