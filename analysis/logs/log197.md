# Log197 — SmallFastDronev1 Indoor Stabilize Hover

**Vehicle:** SmallFastDronev1 (BF_X quad, V4.6.3v2-SFD e0cb6544)
**Mode:** Stabilize
**Duration:** 54 seconds (armed 58.4s, takeoff 62.7s, land 116.3s, disarm 122.0s)
**Purpose:** Stable hover for baro thrust compensation calibration

## Key Parameters

| Parameter | Value | Notes |
|-----------|-------|-------|
| BARO1_THST_SCALE | 0 | No compensation |
| EK3_RNG_USE_HGT | 3.0 | Terrain offset mode |
| PSC_POSZ_P | 0.3 | Very low |

## Flight Conditions

- Indoor, level surface
- Hover throttle: 0.082 ± 0.003 (very steady)
- Hover altitude: 0.52–0.81m (rangefinder)
- Vibration: VibeZ ≈ 3.2 m/s² RMS (consistent throughout)

## Key Findings

### 1. Baro Thermal Drift

Baro temperature dropped 8.5°C (40.5°C → 32°C) from prop airflow cooling during the
54-second flight. Baro error grew progressively:

| Time | Baro Error | Temp Change | Thermal-Corrected Error |
|------|-----------|-------------|------------------------|
| 70–75s | +0.078m | -3.2°C | +0.35m |
| 95–100s | -0.478m | -7.0°C | +0.13m |
| 105–110s | -0.540m | -8.0°C | +0.15m |

### 2. BARO1_THST_SCALE Estimation

Multi-variable regression separating thermal and propwash:
```
baro_error = 11.294 * throttle + 0.0864 * temp_change + (-0.732)
R² = 0.30 (poor — throttle barely varied)
```

Direct computation from thermal-corrected hover error:
- Propwash-only baro error at hover: +0.13 to +0.15m (baro reads slightly high)
- Implied THST_SCALE: **-19 to -22 Pa** (propwash decreases pressure at hover altitude)
- Very small — thermal drift dominates by 4x

### 3. Ground Effect at Motor Start

BAlt dropped from +0.17m to **-6.38m** when motors started — massive positive pressure
at ground level. This is the opposite sign from hover-altitude propwash (ground effect
increases pressure, hover propwash slightly decreases it).

### 4. Vibration Rectification Factor (VRF)

Regression of IMU AccZ offset vs IMU temperature during hover:
```
AccZ_offset = VRF + thermal_coeff * temp_change
VRF = +0.089 m/s²  (R² = 0.82)
Thermal coeff = -0.045 m/s²/°C (residual after TCAL)
```

### 5. TCAL Upside-Down Calibration Discovery

The INS_TCAL1_ACC1_Z coefficient (55844) is 1861x larger than X-axis and 41x larger
than Y-axis. Analysis revealed the board was calibrated upside down:

| | Drift rate | Drift over flight |
|---|---|---|
| Raw sensor (no TCAL) | -0.014 m/s²/°C | 0.09 m/s² |
| TCAL overcorrection | -0.053 m/s²/°C | 0.35 m/s² |
| Net (what we see) | -0.067 m/s²/°C | **0.45 m/s²** |

**Root cause:** Scale factor drift (0.34%/°C) reverses sign when orientation flips.
Upside-down cal learns `offset + scale×g`; right-side-up flight has `offset - scale×g`.
Error = 2 × scale × g = 0.067 m/s²/°C.

### 6. EKF Z-Bias Evolution

XKF2 AZ bias changed from -0.510 (ground) to -0.201 (late hover) — a shift of +0.309.
This tracked the combined VRF (+0.089) and TCAL overcorrection (+0.359) at 69% gain,
with the remainder absorbed into position/velocity corrections.
